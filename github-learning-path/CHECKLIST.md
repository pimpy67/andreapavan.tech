# Checklist — Organizzazione GitHub

## FASE 1 — Crea il repo indice (fai subito, 5 minuti)

1. Vai su https://github.com/andreapavan-ops
2. Clicca **New repository**
3. Nome: `learning-path`
4. Descrizione: `Esercizi e appunti del mio percorso di sviluppo web, organizzati per linguaggio e difficoltà`
5. Visibilità: **Public**
6. Spunta **Add a README file**
7. Clicca **Create repository**
8. Apri il README appena creato, clicca la matita (modifica) e incolla il contenuto di `github-learning-path/README.md` che trovi in PORTFOLIO_5

---

## FASE 2 — Sposta i progetti su pimpy67 (fai quando hai tempo, uno alla volta)

Per ogni repo da spostare:

1. Apri il repo su `andreapavan-ops` (es. `gestionale-libri`)
2. Vai in **Settings** (tab in alto a destra)
3. Scorri fino in fondo → sezione **Danger Zone**
4. Clicca **Transfer repository**
5. Digita il nome del repo per confermare
6. Nel campo "New owner" scrivi: `pimpy67`
7. Conferma il trasferimento

> ⚠️ Dopo il trasferimento il link vecchio viene reindirizzato automaticamente per un po', poi smette. Aggiorna il README di learning-path se hai link diretti.
